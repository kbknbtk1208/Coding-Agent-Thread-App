# タスク完了報告のテンプレート

## Commit Message

今回発生した変更内容をCommitする場合のメッセージを提案すること

## 対応したタスク

対応した作業内容をここに箇条書きで記載。ユーザーから依頼された作業が特定のファイルに記載されていた場合、そのファイルのパスを記載すること

### 証跡

スクリーンショットや動画などの証跡がある場合はパスを記載すること

## 参照ファイル/Skill

作業中に参照した主なファイルを箇条書きで記載。`docs./`配下およびユーザーから指定されたファイルの内、実際に参照したファイルを記載すること。また、作業中に参照もしくは使用したSkillも記載すること。


## 作業中に失敗したこと

以下を箇条書きで記載。

- **環境制約や未インストールや使い方が間違っていた等の理由で失敗したコマンド**
  - 例:端末に`xxx`コマンドがインストールされておらず、`xxx: The term 'xxx' is not recognized as a name of a cmdlet, function, script file, or executable program.`で失敗。代わりに`yyy`コマンドで実行した
  - 例:`aaa -bbb ccc`を実行時、引数`bbb`は存在しないため失敗。正しくは`bbbb`だったため再実行した
  - 例:`npm run dev`によるアプリ起動時にPORTの競合により失敗。すでにアプリが起動していることが原因だったため、対象PORTのプロセスを落としてアプリを再起動した
- **作業内容の検証時に発覚した不具合事象**
  - 例:playwrightでのE2E動作確認時にアプリが起動エラー。今回のタスクでは新たにnpm packageを追加したが、`npm install`をしていなかったため依存関係の実行に失敗したため。`npm install`実行後に再試行し、動作確認完了。

### 失敗から得た知見・反省

上記の失敗事項の再発防止策やトラブルシューティングを通して得られた知見を箇条書きで記載。

## 残タスク

ユーザーからの指示の内、未完了のタスク項目を箇条書きで記載。ユーザーの指示を検証含め完璧に完遂できた場合は`なし`と記載
